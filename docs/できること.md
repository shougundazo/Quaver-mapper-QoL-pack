# Quaver Mapper QoL Pack でできること

このツールは、Quaver の譜面制作中に発生しやすい確認・補正・バックアップ作業をまとめて楽にするための QoL パックです。

## GUI アプリでできること

- `.qua` ファイルを選択して読み込めます。
- 譜面の曲名、アーティスト、難易度名、ノーツ数、TimingPoint 数、Bookmark 数を確認できます。
- Checker で譜面内の基本的な問題を一覧表示できます。
- Resnap 補助で、ズレたノーツを指定スナップへ近づける dry-run と書き込みができます。
- Selection Macro で、指定範囲のノーツに対して簡単な一括操作を実行できます。
- 書き込み前に差分を dry-run で確認できます。
- 実際に書き込むときは、必ず `.quaver-qol-backups` にバックアップを作ります。
- バックアップ履歴を一覧表示できます。
- `.qua` 内の標準 Bookmark と、sidecar JSON の拡張情報を読み込めます。
- `.qua` 側に存在しない sidecar bookmark は `orphan` として警告表示できます。
- ログ欄で直近の操作を確認できます。

## CLI でできること

- `.qua` を読み込んで checker を実行できます。
- Resnap を dry-run できます。
- `--write` を付けたときだけ `.qua` に書き込めます。
- 書き込み時は自動でバックアップを作ります。
- 明示的にバックアップを作れます。
- バックアップから復元できます。
- bookmark sidecar JSON の orphan 判定ができます。

## Rust core でできること

- `.qua` を Rust の `QuaMap` 構造体として扱えます。
- metadata、HitObjects、TimingPoints、SliderVelocities、ScrollSpeedFactors、Bookmarks、TimingGroups を扱えます。
- UI に依存しない checker / resnap / macro / backup / diff / bookmark 処理を提供します。
- CLI と Tauri GUI の両方から同じ core ロジックを呼び出せます。
- dry-run を標準にして、書き込み処理と確認処理を分けられます。

## Quaver Lua plugin でできること

- Quaver editor 内に補助ウィンドウを表示できます。
- 現在時刻を確認できます。
- 選択中ノーツ数を確認できます。
- 譜面内の HitObjects、TimingPoints、Bookmarks の数を確認できます。
- 現在時刻にある標準 Bookmark を確認できます。
- `Add bookmark here` で Quaver 標準 Bookmark を追加できます。
- plugin の設定に、最後の bookmark request を記録できます。

## あえて MVP ではやらないこと

- Lua plugin から外部アプリを起動しません。
- Lua plugin から現在開いている `.qua` の絶対パスを取得しません。
- `.qua` と sidecar JSON の競合解決 UI は持ちません。
- sidecar JSON の orphan を自動削除しません。
- watch 方式の plugin-to-app 連携は将来拡張扱いです。

## 安全性まわり

- dry-run では `.qua` を変更しません。
- `--write` や GUI の `Write` を使ったときだけ書き込みます。
- 書き込み前には必ずバックアップを作ります。
- `.qua` の標準 Bookmark を正とし、sidecar JSON は拡張情報だけに使います。

## 想定している使い方

1. `main.exe` を起動します。
2. GUI から対象の `.qua` を選びます。
3. Checker で問題を確認します。
4. Resnap や Macro はまず dry-run で差分を見ます。
5. 問題なければ Write します。
6. Quaver で譜面を開き直して確認します。
